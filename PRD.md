# **Product Requirement Document (PRD): Resume Screener**

---

## **1. Executive Summary (The "Why")**

### **1.1 Problem Statement**
Recruiters initially spend a lot of time on the resume screening process, and the ATS available in the market are quite expensive.

### **1.2 Goals**
*   **Goal 1:** Reduction Cost Efficiency and Screening Time.
*   **Goal 2:** Standardized & Unbiased Evaluation.
*   **Goal 3:** Instant Visibility into Skill Gaps.
*   **Goal 4:** Improve recruiter decision making.
*   **Goal 5:** Structure and transparency candidate scoring.
*   **Vision:** I want to reduce initial resume processing time and help recruiters find good candidates in less time and at a lower cost; improve recruiter decision making efficiency by providing structure and transparency candidate scoring.

### **1.3 Objectives**
*   **Objective 1: Screening Efficiency**
    *   Transform the manual screening process into an instantaneous AI-driven workflow to drastically reduce processing time and handle high volumes effortlessly.
*   **Objective 2: Quality Filtering**
    *   Optimize the talent pipeline by automatically filtering out irrelevant profiles, ensuring recruiters focus only on high-potential candidates.
*   **Objective 3: Precision Skill Detection**
    *   Ensure 100% accuracy in identifying "Must-Have" skills by cross-referencing resume data with Job Description (JD) requirements.
*   **Objective 4: Accelerated Decision Making**
    *   Enable fast-track 'Select/Reject' decisions by providing recruiters with a smart dashboard that visualizes skill gaps and candidate strengths using a fully transparent and structured scoring model.

## **2. User Flow & Workflow Breakdown**

### **Phase 1: Authentication & Subscription Flow**
**Flow:** Login ➔ Sign-up ➔ Dashboard (Lock) ➔ Choose Plan ➔ Payment Gateway ➔ Success Redirect ➔ Full Dashboard Access

**Workflow Breakdown:**
*   **Authentication:** User completes Login or Sign-up to access the platform.
*   **Restricted Access:** User lands on a "Locked Dashboard" where AI features are visible but disabled.
*   **Plan Selection:** User clicks "Upgrade" and chooses a subscription plan (Basic/Pro).
*   **Secure Payment:** User is redirected to a Payment Gateway to complete the transaction.
*   **Success Redirect:** After payment, the user is redirected to a "Payment Successful" confirmation page.
*   **Full Access:** The system automatically unlocks all AI screening features and updates the user's credits.

### **Phase 2: Bulk Upload & Processing**
**Flow:** New process Start ➔ JD Fill ➔ Upload Files (Bulk) ➔ AI Engine Start (Background) ➔ Progress Bar (Live Update) ➔ Completed ➔ showDashboard

**Workflow Breakdown (Step-by-Step):**
*   **Initiate Process:** The recruiter clicks on the "Start New Screening" button to create a new hiring campaign.
*   **Job Configuration (JD Fill):** The user inputs the Job Description (JD) and defines "Must-Have" vs. "Nice-to-Have" skills to set the AI's evaluation logic.
*   **Bulk File Upload:** The system allows the user to drag and drop multiple resumes (PDF, Docx) simultaneously.
*   **System Validation:** Before processing, the system checks for file corruption, duplicates, and verifies if the user has enough "Screening Credits."
*   **Asynchronous AI Processing:** The AI engine runs in the background to parse text, identify skills, calculate experience, and generate a match score.
*   **Real-time Progress Tracking:** A visual progress bar updates live, showing the percentage of resumes processed.
*   **Auto-Redirection:** Once the progress reaches 100%, the system automatically redirects the user to the Insights Dashboard to view results.

### **Phase 3: Evaluation & Intelligence (The Dashboard)**
1.  **Ranked Table:** View candidates automatically sorted by AI Match Score.
2.  **Search & Filter:** Instantly find candidates using the Search Bar or Filter by experience, location, or specific skills.
3.  **Quick Scan:** Identify Missing Skills (highlighted in red) and score breakdown at a glance.
4.  **Right Sidebar Deep-Dive:** Click a row to open a sidebar showing the candidate’s Full Profile.
5.  **AI Insights:** Review summarized Projects & Impact generated by AI.
6.  **Collaborative Notes:** Add Call Notes and internal feedback directly to the profile.
7.  **Final Action:** Execute a Select or Reject decision with one click.

### **Phase 4: Decision & Outcome**
1.  **Status Update:** (Shortlist / Hold / Reject / Select).
2.  **Export:** Final List to CSV/Excel.

## **3. Functional Requirements**

### **3.1 User Management & Billing**
*   **FR 1.1 (Authentication):** The system shall allow users to create and manage accounts via email/password and OAuth integrations (Google/LinkedIn).
*   **FR 1.2 (Subscription Plans):** The system shall support a multi-tier subscription model (Free, Basic, Pro) with configurable limits on resume processing volume.
*   **FR 1.3 (Payment Processing):** The system shall integrate with secure payment gateways (Stripe/Razorpay) to handle one-time credits and recurring billing.
*   **FR 1.4 (Access Control):** The system shall enforce feature-gating, restricting AI screening and bulk uploads to users with active subscriptions or sufficient credits.

### **3.2 Job Configuration (JD Setup)**
*   **FR 2.1 (JD Input):** The system shall provide a text input interface and file upload option for Job Descriptions (JD).
*   **FR 2.2 (Skill Weighting):** The system shall allow recruiters to categorize skills as "Must-Have" (Mandatory) or "Nice-to-Have" (Preferred) for weighted scoring logic.
*   **FR 2.3 (Selection Criteria):** The system shall allow users to set hard filters for minimum years of experience and specific education levels.

### **3.3 Resume Ingestion & AI Processing**
*   **FR 3.1 (Bulk Upload):** The system shall support concurrent bulk uploads of resumes in PDF and DOCX formats.
*   **FR 3.2 (Data Extraction):** The AI engine shall automatically parse and extract Name, Contact Details, Skills, Experience, and Education from uploaded resumes.
*   **FR 3.3 (Fraud & Keyword-Stuffing Detection):** The system shall detect "white-fonting" (hidden text used to trick AI) and flag resumes that use invisible keywords to artificially inflate match scores.
*   **FR 3.4 (Progress Monitoring):** The system shall provide a real-time progress bar displaying processing percentage and Estimated Time of Completion (ETC).

### **3.4 Scoring & Intelligence Engine**
*   **FR 4.1 (Match Scoring):** The system shall generate a "Match Score" (0–100%) based on the semantic overlap between the resume and the JD requirements.
*   **FR 4.2 (Skill Gap Analysis):** The system shall identify and visualize "Missing Skills" by comparing the candidate's profile against the mandatory JD requirements.
*   **FR 4.3 (AI Summarization):** The system shall generate a concise summary (2-3 sentences) focusing on the candidate's professional impact and key projects.
*   **FR 4.4 (Institutional Tiering):** The system shall categorize candidates into Tier levels (T-1, T-2, T-3) based on the global/national ranking of their colleges and the reputation of their previous employers.
*   **FR 4.5 (Score Breakdown):**

*   **FR 4.6 (Links):** Find links (LinkedIn and GitHub) and attach in sidebar.

### **3.5 Recruitment Dashboard (UI)**
*   **FR 5.1 (Ranked Display):** The system shall present candidates in a table view, automatically sorted by AI Match Score, with key data (Tier level, Experience) visible at a glance.
*   **FR 5.2 (Search & Filtering):** The system shall provide a real-time search bar and filters for Match Score, Tier level, Experience years, and Location.
*   **FR 5.3 (Detail Sidebar):** The system shall feature a slide-out sidebar that displays the full parsed profile, AI insights, and skill gaps when a candidate row is selected.
*   **FR 5.4 (Visual Indicators):** The system shall use color-coded badges (e.g., Red for missing skills, Green for found skills) for rapid visual assessment.
*   **FR 5.5 (Collaborative Notes):** The system shall include a timestamped notes section for recruiters to log feedback and interview observations.

### **3.6 Decision & Export**
*   **FR 6.1 (Status Management):** The system shall allow recruiters to update a candidate’s status (Shortlisted, On Hold, Rejected, Selected) with a single click.
*   **FR 6.2 (Data Export):** The system shall allow users to export selected or all candidates' data into downloadable CSV or Excel formats for external use.
*   **FR 6.3 (Bulk Actions):** The system shall allow users to select multiple candidate rows to update status or delete records simultaneously.

## **4. Non-Functional Requirements (The "Bones")**

| Category | Requirement Description |
| :--- | :--- |
| **Performance** | AI must parse and score each resume in < 5 seconds; UI response time for filtering < 200ms. |
| **Security** | End-to-end encryption (AES-256) for candidate data; PCI-DSS compliance for payments. |
| **Availability** | Target 99.9% system uptime with automated daily data backups. |
| **Scalability** | Support for simultaneous bulk uploads of up to 10,000 resumes across the platform. |
